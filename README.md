# B站疑似矩阵号标记 — 油猴脚本

## 功能简介

在 B 站全站（视频页、动态页、直播页、搜索页等）自动标记疑似矩阵号。

提供两套独立的匹配规则：

| 规则类型 | 匹配范围 | 命中动作 |
|----------|----------|----------|
| **用户名规则** | 评论区用户名、视频卡片作者、动态发布者、UP主名称等 | 保留原用户名 + 可配置前后缀，底部颜色虚线标记 |
| **评论内容规则** | 评论正文 | 左侧颜色边框 + 淡色背景高亮，匹配文字 `<mark>` 高亮 |

每套规则支持 **普通模式**（关键字包含匹配）和 **正则模式**（正则表达式匹配），通过面板切换开关控制。

此外还提供三大进阶功能：

- **UID 记录**：右键用户名手动确认矩阵号，该 UID 在任意评论区出现自动标记
- **AI 分析**：右键评论调用 DeepSeek AI 自动提取识别关键字和正则
- **每规则独立配置**：每个屏蔽词可独立设置颜色、前后缀、折叠（评论）

## 安装步骤

1. 确保浏览器已安装 **Tampermonkey** 扩展
   - [Chrome 版](https://chrome.google.com/webstore/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo)
   - [Edge 版](https://microsoftedge.microsoft.com/addons/detail/tampermonkey/iikmkjmpaadaobahmlepeloendndfphd)
   - [Firefox 版](https://addons.mozilla.org/firefox/addon/tampermonkey/)

2. 打开 Tampermonkey 管理面板 → 点击 **「新建脚本」**

3. 将 `bili-block-matrix-account.user.js` 文件内容**全部复制粘贴**到编辑器中

4. 按 `Ctrl+S` 保存

5. 打开任意 B 站页面，右上角即可看到 **粉色「疑似矩阵号标记」面板**

## 使用说明

### 面板概览

脚本启动后，页面右侧会出现一个粉色竖条小按钮「矩阵标记」，点击即可展开面板。面板结构如下：

```
┌─────────────────────────────┐
│ ✕ 疑似矩阵号标记     ← 标题栏（点击×收起面板）  │
├─────────────────────────────┤
│ [用户名规则] [评论内容规则]  ← 标签页切换        │
├─────────────────────────────┤
│ 模式：[普通 ○─● 正则]       ← 普通/正则模式切换  │
├─────────────────────────────┤
│ ┌─────────┐ ┌──────────┐   │
│ │ 关键字  ×│ │ 关键字  ×│   ← 当前规则列表       │
│ └─────────┘ └──────────┘   │
│ [________输入框_______] [添加]                     │
├─────────────────────────────┤
│ 已替换 12 个用户名            │
│ 已标记 8 条评论               ← 统计计数           │
└─────────────────────────────┘
```

> 面板默认收起为右侧小按钮，点击展开，点击标题栏「×」收起，避免遮挡页面内容。

### 用户名规则

1. 切换到 **「用户名规则」** 标签页
2. 在输入框中输入关键字（如 `电影`），点击「添加」或按回车
3. 页面中所有**用户名包含"电影"**的账号，保留原用户名显示，底部加颜色虚线标记。可通过该规则的独立前后缀设置追加文本（如 `[广告]电影解说[标记]`）
4. 每个规则标签左侧有**颜色圆点**，点击规则文本可展开内联设置，独立配置该规则的颜色/前后缀

### UID 记录

除了按关键字匹配，还可以**手动确认并记录**特定账号为矩阵号，之后该账号在任何评论区出现都会自动标记：

1. 在评论区**右键**任意用户名，选择「记录为矩阵号」
2. 脚本自动从用户名链接中提取 UID 并记录（提示「已记录 UID: xxx」）
3. 此后该 UID 在任意视频/动态的评论区出现，用户名后会显示可配置标记文本（默认 `(有疑似矩阵记录)`），橙色虚线标记，区别于关键字命中（粉色系）
4. UID 命中**优先于**关键字命中——同时满足时显示 UID 标记

管理已记录的 UID：在「用户名规则」标签页底部展开「UID 记录」区，可查看/删除已记录 UID、修改标记文本。

### 评论内容规则

1. 切换到 **「评论内容规则」** 标签页
2. 输入关键字（如 `加群`），添加规则
3. 评论正文中包含"加群"的评论，左侧出现竖条边框和淡色背景，匹配到的"加群"二字会用 `<mark>` 标签高亮显示
4. 每个规则标签左侧有**颜色圆点**，点击规则文本可展开内联设置，独立配置该规则的颜色/折叠/前后缀

### AI 分析

右键任意评论可调用 DeepSeek AI 自动提取识别关键字和正则表达式，无需手动分析：

1. 在面板「AI 设置」区配置 **API Key**（默认为空，需自行申请）和模型名（默认 `deepseek-v4-flash`）
2. 右键任意评论，选择分析范围：
   - **AI分析（评论+用户名）**：分析当前评论文本 + 发布者用户名
   - **AI分析此用户所有评论**：收集该用户在当前页的所有评论一起分析
   - **AI分析整个评论区**：分析整个评论区，找出可疑的矩阵号评论模式
3. AI 返回后弹出结果窗口，分「关键字（普通匹配）」和「正则表达式」两组，勾选要添加的项目
4. 点击「添加为规则」，勾选项自动添加为评论规则并立即生效

> AI 提取的关键字作为普通规则（isRegex=false），正则作为正则规则（isRegex=true），支持混合匹配。

### 每屏蔽词独立配置

每个屏蔽词（规则）都可以拥有**独立的设置**，不再共用全局配置：

- **点击规则标签的文字部分**，会在该标签下方内联展开设置行
- 设置项：高亮颜色（12 预设色 + 原生取色器）、折叠隐藏（仅评论）、前缀文本、后缀文本
- 同一时间只展开一个规则的设置，点击其他规则会自动收起
- 改动任意设置后即时生效（清除旧标记重新处理）

预设颜色（12 种）：粉、红、橙、黄、绿、青、蓝、靛、紫、灰、棕、黑

### 默认设置（新建规则继承）

评论标签页底部的「默认设置」折叠区用于配置**新建规则的默认值**。已有规则的设置不受此区域影响，需点击对应规则标签单独修改。

| 选项 | 说明 |
|------|------|
| **高亮颜色** | 新建规则继承的默认颜色。提供颜色选择器和 12 种预设色 |
| **折叠隐藏** | 新建规则是否默认开启折叠 |
| **前缀文本** | 新建规则继承的默认前缀（如 `[广告]`） |
| **后缀文本** | 新建规则继承的默认后缀（如 `[已标记]`） |

所有设置改动后会自动保存并清除旧标记重新处理，即时生效。

### 正则模式

1. 将模式开关切换到 **「正则」**
2. 直接输入正则表达式语法（不需要 `/ /` 包裹），例如：
   - `\d+号` — 匹配"123号"、"10号"等
   - `影[视剧]` — 匹配"影视"或"影剧"
   - `价.{2}` — 匹配"价格"、"价位"等
3. 如果输入了无效的正则语法，该条规则会被跳过，不影响其他规则执行

### 删除规则

点击规则标签上的 **×** 即可删除该条规则。已标记的元素会保持标记状态，重新加载页面后不再匹配。

## 常见问题

**Q: 为什么某些用户名没有被标记？**

A: B 站新版评论区采用 Web Components + Shadow DOM，脚本通过递归穿透 Shadow 边界查找用户名与评论元素。若仍有个别未命中，可能是 B 站改版导致内部结构变化，需更新选择器。另外脚本设有 2 秒兜底扫描，滚动加载的新评论会在短时间内自动处理。

**Q: 添加规则后没有立即生效？**

A: 规则添加后会触发一次扫描（约 200ms 防抖），Shadow DOM 内的内容最多再等 2 秒兜底扫描即可标记。若始终无效，请确认关键字拼写与匹配模式（普通/正则）是否正确。评论文本通过递归穿透 Shadow DOM 读取，确保能获取 `bili-rich-text` 等子组件内部的真实文本。

**Q: 删除规则后，已标记的元素能恢复吗？**

A: 已标记的元素会保持标记状态。刷新页面后，因为规则已删除，这些元素将恢复原始显示。

**Q: 面板里的「已替换/已标记」计数是什么意思？**

A: 计数显示的是**当前页面中已标记元素的总数**，会随每次扫描（添加/删除规则、切换模式、切换页面、滚动加载新内容）实时刷新。已标记的元素不会重复计数，因此多次操作后数字仍与页面实际标记数一致。

**Q: 正则模式下的语法要注意什么？**

A: 直接输入正则表达式主体即可，不要加 `/ /` 包裹。例如输入 `\d+` 而不是 `/\d+/`。特殊字符如 `\` 需要转义时正常书写。

**Q: 多个关键字是"且"还是"或"关系？**

A: 是**或**关系。只要用户名/评论匹配任意一条规则即触发标记。

**Q: 如何备份我的规则？**

A: 规则存储在 Tampermonkey 的本地存储中，清除浏览器数据时会丢失。如需备份，可在 Tampermonkey 中导出脚本数据。
